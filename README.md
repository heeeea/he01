# 贺01 AI 实战资源库

这是一个纯前端静态网站，用来展示 AI 工具导航、资源下载、实战教程、作品案例，并承接私域与合作转化。

当前版本已经支持列表页、详情页和本地轻后台。前台内容从 `data/` 目录里的 JSON 读取，`admin.html` 可编辑内容并导出 JSON。

## 页面结构

- `index.html`：首页
- `tools.html`：AI 工具导航
- `resources.html`：资源下载列表
- `resource-detail.html`：资源详情页，访问方式：`resource-detail.html?id=资源id`
- `tutorials.html`：实战教程列表
- `tutorial-detail.html`：教程详情页，访问方式：`tutorial-detail.html?id=教程id`
- `cases.html`：作品案例列表
- `case-detail.html`：案例详情页，访问方式：`case-detail.html?id=案例id`
- `contact.html`：联系合作
- `admin.html`：本地资源库管理台
- `style.css`：全站样式
- `script.js`：前台交互、列表渲染和详情页渲染
- `admin.js`：后台交互、localStorage、导入导出

## 详情页说明

三个详情页都通过 URL 参数里的 `id` 查找 JSON 数据。找不到内容时会显示友好提示，例如“资源不存在或正在整理中”。

资源详情页展示资源标题、类型、状态、更新时间、适合人群、格式、文件大小、简介、详细说明、使用步骤、下载入口、注意事项、FAQ、相关资源和联系合作入口。

教程详情页展示教程标题、分类、难度、阅读时间、更新时间、状态、简介、摘要、正文、操作步骤、FAQ、相关资源、下一步建议和联系合作入口。

案例详情页展示项目名称、状态、分类、适合人群、更新时间、简介、背景、问题、核心功能、实现过程、截图、升级方向、结果、经验总结和联系合作入口。

## JSON 字段说明

`data/resources.json` 每条资源支持：

- `detailContent`：详细说明
- `usageSteps`：使用步骤数组
- `notes`：注意事项数组
- `faq`：常见问题数组，格式为 `{ "question": "问题", "answer": "回答" }`
- `relatedResourceIds`：相关资源 id 数组

`data/tutorials.json` 每篇教程支持：

- `detailContent`：教程正文，支持换行显示
- `steps`：操作步骤数组，格式为 `{ "title": "步骤标题", "description": "步骤说明" }`
- `faq`：常见问题数组
- `relatedResourceIds`：相关资源 id 数组
- `nextActions`：下一步建议数组

`data/cases.json` 每个案例支持：

- `background`：项目背景
- `process`：实现过程数组，格式为 `{ "title": "阶段标题", "description": "阶段说明" }`
- `upgrades`：后续升级方向数组
- `relatedResourceIds`：相关资源 id 数组
- `result`：项目结果
- `lessons`：经验总结

## 后台编辑详情内容

`admin.html` 是本地静态维护工具，不是真正安全的在线后台。

使用流程：

1. 打开 `admin.html`。
2. 在站点设置、AI 工具、资源、教程、案例里添加或修改内容。
3. 资源管理可编辑详细说明、使用步骤、注意事项、FAQ、相关资源 id。
4. 教程管理可编辑教程正文、操作步骤、FAQ、相关资源 id、下一步建议。
5. 案例管理可编辑项目背景、实现过程、后续升级、项目结果、经验总结、相关资源 id。
6. 点击导出按钮，导出对应 JSON。
7. 用导出的 JSON 替换项目 `data/` 文件夹里的同名文件。
8. 重新部署网站，前台就会显示新内容。

后台保存使用浏览器 `localStorage`。换电脑、换浏览器或清理浏览器数据后会丢失，所以要定期导出 JSON 备份。

## 如何添加完整资源

1. 进入“资源下载管理”，点击“新增资源”。
2. 填写资源名称、类型、简介、适合谁用、格式、文件大小、下载链接、相关教程链接、标签、状态、更新时间。
3. 填写详细说明。
4. 使用步骤和注意事项每行一条。
5. FAQ 使用 `问题 | 回答`，每行一组。
6. 相关资源 id 用逗号分隔。
7. 保存后导出 `resources.json`，替换 `data/resources.json`。

## 如何添加完整教程

1. 进入“教程管理”，点击“新增教程”。
2. 填写教程标题、分类、难度、阅读时间、简介、内容摘要、标签、状态、更新时间。
3. 填写教程正文，换行会在详情页保留。
4. 操作步骤使用 `步骤标题 | 步骤说明`，每行一组。
5. FAQ 使用 `问题 | 回答`，每行一组。
6. 相关资源 id 用逗号分隔，下一步建议每行一条。
7. 保存后导出 `tutorials.json`，替换 `data/tutorials.json`。

## 如何添加完整案例

1. 进入“案例管理”，点击“新增案例”。
2. 填写项目名称、分类、状态、适合人群、解决问题、核心功能、项目简介、截图链接、标签、更新时间。
3. 填写项目背景。
4. 实现过程使用 `阶段标题 | 阶段说明`，每行一组。
5. 后续升级方向每行一条。
6. 填写项目结果、经验总结、相关资源 id。
7. 保存后导出 `cases.json`，替换 `data/cases.json`。

## 本地预览

推荐用本地服务器预览：

```bash
python -m http.server 4173
```

然后打开：

```text
http://127.0.0.1:4173/
```

直接双击 HTML 用 `file://` 打开时，部分浏览器会限制读取本地 JSON，网站会自动使用内置示例数据兜底。

## 部署

这是纯静态项目，不需要构建命令。Vercel、Cloudflare Pages、GitHub Pages 都可以直接部署根目录。

## 当前限制

- `admin.html` 没有登录和权限控制，不适合作为公开安全后台。
- 不支持真正在线保存，所有编辑先保存在本机 `localStorage`。
- 不支持真实文件上传，资源下载建议先使用网盘、GitHub、官网链接等外链。
- 搜索按钮仍是占位功能。

如果正式上线时担心别人访问 `admin.html`，可以不上传 `admin.html` 和 `admin.js`，只上传前台页面、样式、脚本和 `data/` 文件。
